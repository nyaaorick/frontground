# Project Specification: Job Seeker Forum (agent.md)

## 1. Project Overview
**Target Audience:** Job seekers in Mainland China.
**Core Concept:** A Reddit-style, text-only discussion platform focused on company reviews, interview experiences, and workplace discussions. 
**Development Strategy:** "Fail-Fast" iteration, Vibe Coding, utilizing Cursor IDE and Prompt Engineering.

## 2. Technology Stack
* **Frontend & Meta-Framework:** Next.js (App Router).
* **Backend & Database:** Supabase (PostgreSQL).
* **CDN & Security:** Cloudflare (Note: Anticipate potential GFW blocking; configure accordingly).
* **Deployment:** US-based server (Optimized for lowest cost/free tiers; accepting higher latency as a trade-off).

## 3. Core Features & Requirements

### 3.1 Content Structure (Reddit-style)
* **Company Blocks (`/r/[company_name]`):** Dedicated discussion zones for specific companies.
* **User Profiles (`/u/[username]`):** Public profile pages displaying a user's public posts and comments.
* **Nested Discussions (`/comment/[id]`):** Support for threaded comments (comments on comments).
* **Media Constraints:** strictly **TEXT-ONLY**. Image uploads are NOT supported in the MVP.

### 3.2 User Privileges & Moderation
* **User Rights:** Users can only create and delete their *own* posts/comments. (No editing in MVP, or restricted editing).
* **Social Features:** Support for Direct Messaging (PMs), leaving profile messages, and adding friends.
* **Content Moderation:** Automated AI-bot review system for all user-generated content to filter spam/inappropriate content.

### 3.3 Search System
* **Fuzzy Search Engine:** Implement intelligent fuzzy search.
* **Ranking Priority:** Search results must prioritize Company blocks (`/r/`) first, followed by User profiles/comments (`/u/`). (Logic similar to Tianyancha/Qichacha entity search).

### 3.4 Data Seeding & AI Integration
* **Cold Start Strategy:** Utilize AI web scrapers to aggregate initial company data and reviews.
* **AI Users:** Create designated AI user accounts with specific permissions to automatically publish seeded posts. 
* **Authenticity Flagging:** AI-generated/scraped posts must maintain source authenticity or be flagged as bot-generated by the AI bot system.

## 4. Scalability & Future Architecture (Phase 2)
* **Subdomain Architecture:** System must be designed to easily spin up subdomains for expanded features.
* **Testing/Assessment Module:** Future integration of career aptitude tests, psychological guidance, and personality assessments.
* **E-commerce/Traffic Routing:** Ability to link social media traffic (e.g., Xiaohongshu) to a built-in storefront for paid assessments.

## 5. Development Guidelines for AI Agent
* **Code Generation:** Default to Next.js 15 best practices (Server Components where applicable, Client Components only when interactivity is needed).
* **Database:** Use Supabase client for data fetching. Write clean, secure RLS (Row Level Security) policies for Postgres.
* **Simplicity:** Do not over-engineer. Follow the "fail-fast" methodology. Build the MVP text-forum routing (`/r/`, `/u/`) and Supabase schema first.
